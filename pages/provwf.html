ProvWF

SURROUND's general-purpose provenance workflow tool

ProvWF is a Python library we've made to create workflows that track what they do using the standardised PROV Data Model (Prov-DM). We use ProvWF for anything we do that we can wrap in Python which includes:

  • Natural Language Processing tasks
  • Machine Learning model traning and use
  • automated calls to our Knowledg Graph products
  • our various APIs' responses to queries
  • custom workflows for projects

ProvWF uses a specialised form of the PROV Data Model to capture all aspects
of its execution: how it runs and what it produces.

When should you use ProvWF?

  1. Custom data pipelines when you want to build custom processing systems and track what they do
  2. Server or Client actions when you want to log server or client actions in a sophisticated way
  3. Machine Learning when you want to track ML training and deployment

Of course! ProvWF pipelines produce provenance information that can be absorbed by our SURROUND Ontology Platform so that you can use SOP to query and visualise ProvWF results.
